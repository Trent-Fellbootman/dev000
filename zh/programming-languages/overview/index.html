    编程语言：一个高层次概述

    在本部分中，我们将讨论编程语言的目的和它们的一般分类。

    什么是编程语言？为什么我们需要它们？

    一般来说，编程语言是计算机的抽象，旨在使设计、表示、构建和理解计算机程序更容易理解。

    注意，编程语言有时以更宽泛的方式定义，将汇编语言也计算在内； 但在本模块中，我们将"编程语言"定义为仅包含现代、"面向人类"的编程语言，如 C/C++、Python、Rust、Java、JavaScript 等； 我们不包括那些不适用于人类使用的语言，如汇编语言和二进制机器码。

    回想一下，上个模块中提到的计算机最低层（最接近硬件）的抽象是指令集架构（ISA）， 但使用 ISA 进行编程对于人类来说非常不直观。 因此，人们决定构建更高层次的抽象，使之更容易理解和使用； 这正是编程语言存在的目的。

    由于编程语言是面向人类友好的高级抽象， 它们的编程模型通常远离实际计算机硬件（与 ISA 相比）。 例如，几乎每个 ISA 都有的寄存器在几乎所有编程语言中都抽象了出来。

    关于寄存器

    在现代 CPU 中，ISA 中定义的寄存器通常不是实际的硬件组件，但它们也不仅仅是抽象。

    CPU 中确实存在 "物理寄存器"。 然而，它们并不直接映射到 ISA 定义的 "体系结构寄存器"。 典型情况是，物理寄存器比体系结构寄存器多得多； 后者通过一种称为 "寄存器重命名" 的技术在运行时以多对多的方式映射到前者。

    编程语言中常见的一些概念与 ISA 中的 "低级实现" 相比如下：

    编程语言概念 ISA 实现 描述
    变量/常量 保存数据的一块内存 表示 "事物"，如字符串 "Hello, World!"、数字 1.22 或者树形数据
    函数 一般是保存 ISA 指令的内存块（在编译语言中） 能够完成某种任务的工具
    结构体 在编译语言中没有直接的表现（在运行时） 结构化数据的定义，比如 "一个学生结构体具有姓名、年龄和成绩"

    编程语言的通用分类

    目前有许多编程语言； 每种语言都有自己的特点、抽象和权衡，并适用于特定的一组用例； 然而，它们之间存在一些共同点， 可以从多个角度将编程语言分为几个大类。

    在本小节中，我们将讨论一些常见的编程语言分类方法以及各类别语言的一般特点。

    执行模型：编译 vs. 解释

    将编程语言分类的一种常见方法是通过执行模型， 执行模型可以是 "编译"，"解释" 或者二者之间的混合。

    编译语言

    从上个部分中我们了解到， 几乎每个计算机程序在运行时（即程序运行时）的执行都归结为在硬件上执行 ISA 定义的指令。

    然而，编程语言是为人类设计的，而不是为计算机硬件设计的； 因此，必须有一种方法将用编程语言编写的软件转换为机器可执行代码， 无论是在程序运行之前还是运行时。

    编译语言选择前者。 在编译语言中，开发人员用编程语言编写的代码（以形成完整的程序或程序的一部分） 在运行程序之前通过称为"编译器"的软件将其转换为机器可执行代码。 然后，该机器代码形式的软件应用程序可以直接在硬件上运行。

    编译语言有一些与之相关的术语：

    • 源代码：开发人员用编程语言编写的代码。
    • 机器代码：可以直接在硬件上运行的代码。
    • 编译：将源代码转换为机器代码的过程。
    • 已编译的应用程序：已经经过编译（从源代码），通常以机器代码形式，并且可以直接在硬件上运行的软件。
    • 二进制可执行文件：由于机器代码通常以二进制形式存在（例如，001100101010111000110）， 可以直接在硬件上运行的编译应用程序通常称为 "二进制可执行文件"。
    • 编译：“编译”这个动作的动词形式。
    • 编译器：执行编译的软件。
    • 编译时：编译执行的时间。
    • 运行时：程序运行的时间。

    反编译

    你可能想知道是否可能将机器代码转换回源代码。 是的，确实可以。 这个过程称为反编译， 有很多反编译器可以进行反编译。

    编译语言的优点有：

    • 快速且高效： 编译语言最大的优势是它们运行速度快且硬件高效。 编译过程确保编译应用程序中的每个指令都以最高效的方式执行有用的操作。 许多人对现代编译器可以执行的广泛优化范围感到惊讶， 从识别和消除源代码中无用的部分， 到以与硬件更好配合的方式 "重写" 源代码。 此外，由于编译应用程序与硬件直接进行交互 而不引入中间层（与解释语言相比）， 它们运行速度更快，内存占用更少。
    • （有时）更好的错误排除： 在许多情况下（例如变量类型，我们将在后面介绍）， 编译语言的静态特性意味着可以在编译时检测到源代码中的许多错误， 而不是在运行时导致用户遇到故障。

    缺点包括：

    • （通常）学习和使用难度较大： 由于编译语言通常更接近硬件， 学习它们需要更大的努力。 使用它们也通常更难， 因为与硬件接近意味着您必须自己管理许多东西（特别是内存）， 而不是将这个责任委托给一些软件（大多数解释语言）。 此外，用编译语言编写相同的应用程序通常需要更多的代码。
    • 较低的灵活性： 编译语言的静态特性意味着在其中执行某些操作较为困难， 例如在运行时修改变量的类型。
    • 硬件特定：由于机器代码是最接近硬件的抽象， 并且不同操作系统通常具有不同的二进制可执行文件格式， 通常无法在多个操作系统上运行编译的应用程序 或在实现不同 ISA 的硬件上运行。 例如，为 Windows 编译的应用程序无法在 Linux 或 macOS 上运行； 针对 x86 ISA 的 PC 应用程序无法在通常实现 ARM ISA 的智能手机上运行。

    一些常用的编译语言有：

    • C/C++：尽管创建于很多年前，C/C++ 仍然是全球最流行的编译语言之一。 它提供了对硬件的极致控制和无与伦比的性能（如果以正确的方式使用）。 然而，它的学习曲线更陡峭， 并且内存管理极具挑战（但使用智能指针和 RAII 等东西可以减轻这种情况）。
    • Rust：一种现代、相对较新的编程语言。 它是世界上最难学的语言之一， 但一旦熟悉它，您将从其内存安全保证以及其现代的语法和设计中受益匪浅，这使其非常直观和简洁。

    解释语言

    与编译语言不同，解释语言选择使用一个名为"解释器"的特殊软件 在运行时实际 "执行" 源代码。 解释器会读取源代码中的每个 "高级" 指令并执行它。

    从一个角度来看，您可以认为源代码是由 "理解它的虚拟硬件"（解释器）直接执行的； 从另一个角度来看，由于解释器的执行最终归结为机器代码， 您可以认为源代码在运行时由解释器动态地转换为机器代码。

    例如，考虑以下 Python 代码：

    a = 125
b = 137
c = a * b
print(f"The result of {a} times {b} is {c}")

    即使您没有编程经验，上述代码应该很容易理解： 它将 125 和 137 相乘，然后将结果打印到控制台（即执行代码的终端）。

    该代码如何由解释器执行？ 解释器实际上会读取并执行每一行：

    1. 第一行 a = 125 通过分配一块内存，将其命名为 a 并将 125 写入其中来执行。
    2. 执行第二行 b = 137 时，会进行类似的操作。
    3. 执行第三行 c = a * b 时，会先读取之前命名为 a 和 b 的内存块， 然后将它们相乘，将结果写入命名为 c 的已分配内存块中。
    4. 执行第四行 print(f"The result of {a} times {b} is {c}") 时，会读取 a、b 和 c， 然后将它们打印到控制台。

    注意

    以上的解释在硬件上的实际发生情况并不完全正确。 Python 代码的实际执行是一个复杂的过程， 考虑到解释器 "理解" 源代码的方式以及 Python 解释器的执行引擎（即 Python 虚拟机）是一个基于栈的系统。

    但是现在，不必对硬件细节过于担忧； 上面的解释提供了关于解释语言提供的抽象的良好说明。

    解释语言的优点有：

    • 其动态特性允许在代码中进行更大的灵活性。 例如，Python 允许您在运行时更改变量的类型（例如，从整数更改为字符串）， 这在编译语言中通常是被禁止的。
    • 它们通常更容易学习和使用。 解释器提供了一种更人性化的计算机抽象，使编程更容易。 在解释语言中编写相同的软件通常会导致比编译语言更短的源代码。 并且在解释语言中，通常无需手动管理内存；解释器会为您处理。
    • 更好的跨平台和跨 ISA 兼容性： 由于解释器为计算机硬件提供了另一层抽象， 使用解释语言编写的软件可以在安装了该语言解释器的任何计算机上运行， 而无论其 ISA 和操作系统如何。

    缺点包括：

    • 运行速度慢且硬件效率低下。 显然，解释器读取、解析和 "模拟执行" 源代码行比直接执行编译后的机器代码慢得多。 实际上，Python 代码可以比等价的 C++ 代码慢 20 倍到 200 倍。
    • （有时）更难调试： 解释语言的灵活性也意味着有时更难调试它们。 在极端情况下，解释语言中的所有错误都是运行时错误， 因为没有对源代码进行编译时分析。 如果一个应用程序中存在 1/1000 的严重错误， 那么在开发该应用程序时很可能无法发现该错误， 而在不幸的用户打开它时会导致运行时错误。

    混合技术

    除了前面介绍的 "纯" 编译语言/解释语言之外， 还有一些技术模糊了二者之间的界限：

    即时编译（JIT Compilation）

    这项技术允许在运行时编译部分源代码。

    JIT 编译提供了一些优势：

    • 该程序可以在安装了 JIT 编译器的任何计算机上运行。
    • JIT 常常比解释性语言更快，因为 JIT 编译器会将源代码编译为机器代码。
    • JIT 编译器有时可以发现比编译语言的 "静态编译器" 更多的可用优化项， 因为它可以访问程序的运行时状态。
    中间语言（IL）和虚拟机（VM）

    一些语言支持将源代码编译为既非人类可读也非硬件可执行的中间表示。 在运行时，该中间表示由一种称为"虚拟机"的软件执行 （请勿将其与可安装操作系统的虚拟机混淆）。 这种方法最著名的示例是 Java 和它的字节码表示。

    这种技术在速度和跨平台兼容性方面提供了一种权衡： 由于源代码被编译为中间表示， 编译器可以进行一些编译时优化； 而且，引入虚拟机使中间表示能够在安装了虚拟机的任何计算机上执行。

    需要注意的是，中间表示并不设计为人类可读， 因为它被期望由编译器生成，而不是由人类开发者编写。

    编程范式：有状态 vs. 无状态

    将编程语言分类的另一种方法是根据它们的编程范式。 基本上，"编程范式" 是一种用于设计和理解计算机程序的思维模式， 或者您可以将其视为计算机的抽象。 大多数编程语言允许您使用不同的编程范式来构建软件， 但其中一些语言可能鼓励某些范式。

    一般来说，编程范式有两个主要的大类：有状态和无状态。

    有状态编程范式 & 面向对象编程 (OOP)

    有状态编程范式采用了状态的概念， 通常与现代的 "面向对象编程" 等效。

    状态是程序执行过程中会发生变化并影响其行为的东西。 通常，这些状态保存在为程序分配的内存块中。 需要注意的是，在有状态编程范式中，程序的行为取决于输入和状态， 即使输入相同，程序的输出也可能不同。

    例如，考虑一个简单的应用程序，有一个按钮和一个保存的数字。 数字从 0 开始；每次按下按钮时，数字加 1，然后显示在屏幕上。 在这种情况下，“数字”是程序的状态， 因为它在按钮被按下时发生变化，并影响屏幕上的显示内容。

    下面的图表说明了有状态编程范式的编程模型：

    graph TD

    input -->|changes| program-state
    input -->|influences| program-behavior
    program-state -->|influences| program-behavior

    无状态编程范式 & 函数式编程

    无状态编程范式通常以（数学意义上的）函数的形式（将输入转换为输出，无任何副作用）来抽象计算机程序； 通常不鼓励或禁止使用状态。 无状态编程范式通常称为"函数式编程"。

    例如，考虑一个计算两个人之间年龄差的程序。 在函数式编程范式中，这样的过程可以抽象为以下图表：

    flowchart LR

    person-A([person-A])
    person-B([person-B])

    person-A-age([person-A-age])
    person-B-age([person-B-age])

    output([output])

    person-A --> get-age --> person-A-age
    person-B --> get-age --> person-B-age
    person-A-age --> calculate-age-difference
    person-B-age --> calculate-age-difference
    calculate-age-difference --> output

    正如您所看到的，处理的每个步骤（无论是 get-age 还是 calculate-age-difference） 的唯一任务都是将其输入转换为输出； 没有副作用（例如，向屏幕打印内容或更改内存块的内容）， 相同的输入总是产生相同的输出。 除了程序输入之外，不存在影响程序行为的状态。

    编程范式和编程语言

    不同的编程范式具有不同的优点和缺点。 例如，函数式编程使开发人员更易于理解程序的行为 （有状态程序的行为更不可预测，特别是在并发时， 因为它取决于输入和状态）， 而有状态的编程范式在构建程序方面更强大。

    因此，大多数软件应用在不同方面使用不同的编程范式， 而不是仅使用一种范式。 一种常见的做法是将程序的高级结构设计为具有状态的对象 （即具有状态的东西）， 然后利用函数式编程来实现这些对象的功能。

    同样，大多数现代编程语言支持多种编程范式。 然而，其中一些语言可能通过设计鼓励或偏向某一种范式。

    例如，支持有状态范式/面向对象编程的一些编程语言：

    • Java：Java 是一种设计用于面向对象编程（OOP）的最流行语言之一。
    • C/C++：C/C++ 是一种多范式语言，对于面向对象编程有很好的支持。
    • Python：Python 也是一种多范式语言。
    • Rust：Rust 是一种多范式语言，主要鼓励函数式编程， 但也支持面向对象编程。

    支持无状态范式/函数式编程的一些编程语言：

    • LISP：LISP 是最早的编程语言之一。 它具有鼓励现代称为函数式编程的概念的独特设计。
    • Rust：如前所述，Rust 是主要鼓励函数式编程的多范式语言。
    • Python：如前所述，Python 也是一种多范式语言，没有对函数式编程或面向对象编程有很强的偏向。

    通用编程语言 vs. 领域特定语言 (DSLs)

    第三种将编程语言分类的方式是根据适用领域。 可以将编程语言分为两大类别：通用编程语言和领域特定语言 (DSLs)。

    通用编程语言

    通用编程语言是为广泛用途而设计的编程语言。 也就是说，您可以使用它们来构建任何任务的软件。 我们之前提到的大多数语言都属于这个类别，包括 C/C++、Python、Rust 和 Java。

    领域特定语言

    领域特定语言（Domain Specific Languages，DSLs）是为特定领域而设计的语言， 如数学、数据库操作、数据分析和人工智能。 严格来说，许多 DSL 并不符合本节中介绍的编程语言定义， 因为它们不是为构建软件而设计的； 但是，如果您把编程语言的定义概括为 "一种使某些事情更容易做到的抽象或模型"， 那么 DSLs 就符合这个定义。

    一些 DSL 示例：

    • SQL：SQL 是为数据库操作而设计的 DSL。
    • Mermaid：Mermaid 是一种 DSL，允许您以文本形式高效地表示图形、流程图、思维导图等。
    • Slint：Slint 是一个 UI（用户界面，如按钮、窗口和滑动条）框架，提供了一种表示 UI 元素的 DSL。
    • Bash：Bash 可以被视为一种使在终端上进行各种操作更容易的 DSL。
    • MATLAB：MATLAB 是一种流行的用于数学的 DSL，支持各种数学概念和操作，如矩阵和矩阵求逆等。

    有趣事实（个人观点）：开发者评价

    个人而言，我认为MATLAB依然是最流行的语言之一，真是个奇迹， 考虑到它那可怕的集成开发环境、上古的语法和反人类的设计 （比如使用小括号来从1开始索引数组，而其他人都使用0和方括号）。

    尽管MATLAB是我认真学习的第一种语言， 但现在它是我最不喜欢和最少使用的语言。 我强烈不建议新的开发者将MATLAB作为他/她的主要语言， 并鼓励当前使用MATLAB的用户切换到Python， 这对他们自己和开源社区的发展都有好处。

    结论

    在本节中，我们讨论了编程语言的概念以及它们的一般分类。

    一般来说，编程语言是计算机的抽象，旨在使设计、表示、构建和理解计算机程序更容易理解。

    编程语言可以根据以下 3 种方式进行分类：

    1. 执行模型分类：
      • 编译语言：源代码在程序执行之前被编译为机器代码。
      • 解释语言：解释器直接执行源代码。
      • 混合技术：即时编译（JIT Compilation）和中间语言（IL）与虚拟机（VM）。
    2. 编程范式分类：
      • 有状态范式和面向对象编程 (OOP)：程序包含具有状态的对象，除了输入之外，还会修改其行为。
      • 无状态范式和函数式编程：程序被抽象为将输入转换为输出的函数，无副作用，不使用状态。
    3. 适用领域分类：
      • 通用编程语言：可用于构建几乎任何任务的编程语言。
      • 领域特定语言 (DSLs)：针对特定领域而设计的编程语言。
